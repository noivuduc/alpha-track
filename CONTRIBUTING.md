Thank you for your interest in contributing. This document explains how to set up your environment, what areas are open for contribution, and how to submit changes.

## Development Setup

### Prerequisites

- Docker and Docker Compose
- Python 3.11+
- Node.js 18+

### 1. Fork and clone

```bash
git clone https://github.com/noivuduc/alpha-track.git
cd alpha-track
```

### 2. Configure environment

```bash
cp .env.example .env
cp backend/.env.example backend/.env
# Edit backend/.env — set SECRET_KEY at minimum
```

### 3. Start databases

```bash
docker compose up -d postgres redis
```

### 4. Install backend dependencies

```bash
cd backend
python3 -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

### 5. Run migrations

```bash
export DATABASE_URL="postgresql+asyncpg://alphatrack:changeme@localhost:5432/alphatrack"
alembic stamp head   # if Docker init.sql already ran
# or: alembic upgrade head  (fresh local Postgres)
```

### 6. Start the backend

```bash
export DATABASE_URL="postgresql+asyncpg://alphatrack:changeme@localhost:5432/alphatrack"
export REDIS_URL="redis://:changeme@localhost:6379/0"
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

### 7. Start the frontend (separate terminal)

```bash
cd frontend
npm install
NEXT_PUBLIC_API_URL="http://localhost:8000/api/v1" npm run dev
```

Or run everything with: `./start.sh`

## Running Tests

```bash
# Backend
cd backend
source .venv/bin/activate
pytest tests/ -v

# Frontend
cd frontend
npm run lint
npx tsc --noEmit
npm run build
```

## Project Structure

See [README.md](README.md#project-structure) for a full map. Key boundaries:

| Area | Location | Notes |
|---|---|---|
| API endpoints | `backend/app/routers/` | Keep handlers thin — delegate to services |
| Business logic | `backend/app/services/` | Pure computation where possible |
| Data fetching | `backend/app/providers/` | Implement `base.py` interface to add a new source |
| Background tasks | `pipeline/pipeline_worker/tasks/` | Follow existing task patterns |
| Frontend pages | `frontend/src/app/` | Next.js App Router |
| UI components | `frontend/src/components/` | Dashboard tabs, charts |
| API client | `frontend/src/lib/api.ts` | Typed, auto-refreshes tokens |

## Adding a New Data Provider

1. Create `backend/app/providers/your_provider.py` implementing the interface in `base.py`
2. Register it in `backend/app/services/data_service.py`
3. Add the API key variable to `backend/app/config.py` and `backend/.env.example`
4. Add a corresponding entry in `backend/app/admin_seed.py` so it appears in the admin dashboard

## Adding a Database Migration

```bash
cd backend
source .venv/bin/activate
export DATABASE_URL="postgresql+asyncpg://alphatrack:changeme@localhost:5432/alphatrack"

# Generate from model changes
alembic revision --autogenerate -m "your_description"

# Review the generated file in alembic/versions/ before committing
alembic upgrade head
```

Always review autogenerated migration files — Alembic occasionally generates spurious changes for TimescaleDB-managed tables.

## Code Style

**Python**
- Follow PEP 8
- Type hints on all function signatures
- Async functions for anything that touches the database or network
- No bare `except:` — catch specific exceptions

**TypeScript / React**
- TypeScript strict mode is enabled — no `any`
- Keep components focused; extract hooks for state logic
- Use the existing `api.ts` client for all API calls

## Pull Request Process

1. Create a branch: `git checkout -b feature/your-feature` or `fix/issue-123`
2. Make your changes and add tests where applicable
3. Ensure all tests pass and the build is clean
4. Push and open a pull request against `main`
5. Fill in the PR template — describe what changed and why
6. A maintainer will review and may request changes

**PR checklist:**
- [ ] Tests pass (`pytest tests/ -v`)
- [ ] No TypeScript errors (`npx tsc --noEmit`)
- [ ] No new secrets or credentials committed
- [ ] Migration included if schema changed
- [ ] `.env.example` updated if new env vars added
